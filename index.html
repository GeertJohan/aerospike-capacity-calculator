Calculate storage capacity requirements/limits for an Aerospike namespace. Memory capacity requires different formula's which are not yet added to this calculator. This calculator does not (yet) calculate: primary index, secondary indexes, replication.

The Blob size and Records numeric fields accept 'k', 'm' and 'b'. e.g.: "1.2b" == "1,200,000,000"

Use at your own risk. This is not an official Aerospike tool! Verify these calculations by doing your own.

More information about capacity planning: aerospike.com/docs/operations/plan/capacity

Source for this calculator (PR's welcome!): github.com/GeertJohan/aerospike-capacity-calculator
